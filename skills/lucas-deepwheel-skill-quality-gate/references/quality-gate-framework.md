# 质量门禁框架

## 四个等级

- L1 本地草稿：结构合规、安全边界清楚、可手动测试。
- L2 安装候选：真实任务 smoke test 通过，新用户介绍可读。
- L3 GitHub 发布候选：README、安装、测试、安全、贡献、CI、示例齐全；本地、GitHub、Actions、版本/发布与安装版状态已对账；能力变化后的公开文案与双语图片已复核。
- L4 高星目标版：价值清楚、上手极短、边界可信、可贡献、可传播。

## 三档能力声明

每个 Skill 必须写清：

```text
已支持
需要工具 / 权限 / 人工复核
暂不承诺
```

## 严重程度

```text
🔴 严重：阻断安装或发布
🟡 警告：定稿前建议修
🔵 建议：发布加分项
```

## 独立产品入口

每个 Skill 都应先作为独立产品成立，而不是为了衔接另一个 Skill 才存在。

必须检查：

```text
用户不使用上游 / 下游 Skill 时，本 Skill 是否仍有独立价值
是否支持用户直接提供文件、文件夹、文本、图片或其他合适输入
是否避免“没有某个关联 Skill 产物就不能开始”的过度绑定
是否明确哪些场景才建议关联 Skill
```

## 关联 Skill 主动提醒

当用户需求明显跨出当前 Skill 边界时，应主动介绍关联 Skill：

```text
当前 Skill 能做什么
关联 Skill 能补齐哪一半能力
完整链路怎么工作
如果未安装，安装或启用的价值是什么
不安装时怎么降级
是否需要用户确认
```

硬规则：

```text
不自动安装
不强制跳转
不为了衔接而衔接
不把关联 Skill 当成唯一入口
```
