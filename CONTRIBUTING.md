# Contributing

## Scope

Keep this project focused on Agent Skill quality gates, publication readiness, safe reporting, and reusable review guidance.

## Change process

1. Create a feature branch.
2. Keep fixtures synthetic and sanitized.
3. Add positive and negative behavior tests for scanner changes.
4. Run version, generic package, Quality Gate-specific, and behavior validation.
5. Open a pull request and explain capability or compatibility changes.
6. Use Rebase merge after review.

Do not commit credentials, private customer data, local absolute paths, complete logs, protected source assets, or generated test residue.

Installation, repository visibility changes, direct pushes to main, Tag, Release, branch deletion, and external publishing require explicit owner confirmation.
